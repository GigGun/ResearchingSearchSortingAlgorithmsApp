Пузырьковая сортировка (Bubble Sort)

Описание: Простой алгоритм сортировки, многократно проходящий по списку, сравнивающий соседние элементы и меняющий их местами, если они находятся в неправильном порядке.

Сложность:

  • В худшем случае: O(n²)
  • В среднем: O(n²)
  • В лучшем случае: O(n) (если массив уже отсортирован)

Особенности:

  • Устойчивая сортировка
  • На месте (не требует дополнительной памяти)
  • Неэффективна для больших массивов

Быстрая сортировка (Quick Sort)

Описание: Алгоритм "разделяй и властвуй", который выбирает опорный элемент, разделяет массив на подмассивы и рекурсивно сортирует их.

Сложность:

  • В худшем случае: O(n²) (если плохой выбор опорного элемента)
  • В среднем: O(n log n)
  • В лучшем случае: O(n log n)

Особенности:

  • Неустойчивая сортировка
  • На месте (в стандартной реализации)
  • Очень эффективна для больших массивов

Сортировка вставками (Insertion Sort)

Описание: Строит отсортированный массив по одному элементу за раз, вставляя каждый новый элемент в правильную позицию.

Сложность:

  • В худшем случае: O(n²)
  • В среднем: O(n²)
  • В лучшем случае: O(n) (если массив уже отсортирован)

Особенности:

  • Устойчивая сортировка
  • На месте
  • Эффективна для небольших массивов

Линейный поиск (Linear Search)

Описание: Последовательный просмотр элементов массива до нахождения искомого.

Сложность:

  • В худшем случае: O(n)
  • В среднем: O(n)
  • В лучшем случае: O(1) (если элемент первый)

Особенности:

  • Работает на любых структурах данных
  • Не требует сортировки
  • Неэффективен для больших массивов

Двоичный поиск (Binary Search)

Описание: Алгоритм поиска в отсортированном массиве, повторно делящий поисковый интервал пополам.

Сложность:

  • В худшем случае: O(log n)
  • В среднем: O(log n)
  • В лучшем случае: O(1) (если элемент в середине)

Особенности:

  • Требует отсортированный массив
  • Гораздо быстрее линейного поиска
  • Работает только с массивами с произвольным доступом
